# Warehouse Management Information System (WMIS)

A professional warehouse management system optimized for cloud environments. This application is built as a serverless platform, utilizing Flask and Vercel's infrastructure to provide real-time inventory tracking, order processing, and visual analytics.

## System Architecture

The application is engineered for high availability and low operational overhead:
- **Frontend**: A single-page application (SPA) serving high-performance dashboard visualizations and process management interfaces.
- **Backend API**: Flask-based RESTful services deployed as Vercel Serverless Functions.
- **Data Tier**: Vercel Postgres (powered by Neon), providing a serverless relational database optimized for dynamic scaling.

## Key Functionalities

- **Inventory Control**: Real-time stock visibility across zones and aisles.
- **Inbound Logistics**: Streamlined stock reception with automated database updates.
- **Outbound Operations**: Order placement system with integrated inventory validation.
- **Reverse Logistics**: Dynamic status management (Available, Locked, Quarantined) for returned or inspected items.
- **Dashboard Analytics**: Comparative visualizations for stock distribution and order trends.

## Transaction Integrity

The system maintains strict data consistency through database-level logic:
- **FIFO Processing**: Inventory is deducted based on First-In-First-Out logic to ensure accurate batch management.
- **Atomic Transactions**: Orders are processed using database triggers to prevent overselling, ensuring that stock levels never fall below zero even under concurrent request conditions.

## Deployment Information

### Database Configuration

1. Initialize a Vercel Postgres (Neon) instance via the Vercel Storage dashboard.
2. Execute the provided `database_postgres.sql` script within the database query console to establish the schema, constraints, triggers, and seed data.

### Environment Implementation

The backend utilizes the `POSTGRES_URL` environment variable automatically generated by the Vercel-Neon integration. No manual configuration of connection strings is required within the codebase.

### Application Deployment

Deploy using the Vercel CLI or via Git integration:
```bash
vercel --prod
```

## API Specification

The REST API manages core warehouse operations via the following endpoints:
- `GET /api/inventory`: Full inventory state retrieval.
- `GET /api/stats`: Aggregated metrics for analytics.
- `POST /api/inbound`: Receipt of new inventory batches.
- `POST /api/order`: Transactional order placement with overselling protection.
- `POST /api/return`: Status update for specified inventory assets.
